Buttons

Three button types based on how they're used, not their Bootstrap class name. See Button Reference for the complete catalog.

Filled (Action Buttons)

Standalone actions. Color differentiates meaning, not visual weight. All peer actions get equal weight.

.btn-info (blue), .btn-primary (blue), .btn-success (green), .btn-warning (orange), .btn-danger (red), .btnstyle3, .savebtn, .resetbtn

Outline (Toggle / Neutral)

Toggle states, selections, cancel/dismiss. Fills with color on hover.

.btn-default, .btn-mini, .zoom-button, .log-filter-btn

Ghost (Tertiary / Inline)

Navigation, inspection, low-stakes actions. Shouldn't compete visually.

.btnsmall, .btn-icon, .btn-link, .btnstyle / .btnstylerev

Why Filled vs Outline?

Standalone action (Add, Update, Delete)Filled — color differentiates meaning, not weight
Toggle inactive / CancelOutline — needs to contrast with filled active state
Navigation / inspectionGhost — shouldn't compete with action buttons
border-radius5px — all buttons, no exceptions
transition0.15s ease on background, color, border-color, filter
filled hoverfilter: brightness(0.85)
outline hoverFill with color, text goes white
ghost hoverrgba(color, 0.1) background
